# Internationalization

`@weave/i18n` is signal-native translation. It's an *optional* add-on — the runtime never depends on it, so an app that doesn't call `createI18n()` pays nothing. When you do use it, translation is just another signal graph: `t('key')` reads the active-locale and message signals, so every `{{ t('…') }}` (and every `computed`/`effect` over `t`) updates the instant `setLocale()` runs — no reload, no streams, no RxJS.

## Setup

Create one instance, usually at startup. By default it becomes the global one, so a bare `import { t } from '@weave/i18n'` resolves to it everywhere:

~~~ts title="app/i18n.ts"
import { createI18n } from '@weave/i18n';

export const i18n = createI18n({
  lang: 'en',
  fallbackLang: 'en',
  messages: {
    en: {
      hello: 'Hello, {{ name }}!',
      cart: { items: '{count, plural, =0 {empty} one {# item} other {# items}}' },
    },
    lt: {
      hello: 'Sveikas, {{ name }}!',
      cart: { items: '{count, plural, =0 {tuščias} one {# prekė} few {# prekės} other {# prekių}}' },
    },
  },
});
~~~

| Config | Meaning |
|--------|---------|
| `lang` | Initial active language |
| `fallbackLang` | Consulted when a key is missing in the active language |
| `messages` | Eagerly-available messages, keyed by language |
| `langs` | Advertised languages for a switcher (defaults to the keys of `messages`) |
| `loader` | Lazy loader for languages/scopes not in `messages` |
| `missing` | Called when a key resolves nowhere (default: returns the key) |
| `global` | Register as the global instance for the bare `t` (default `true`) |

## Translating

`t(key, params?)` looks up a (dotted or flat) key and interpolates params. The same `t` works in templates **and** in plain `.ts`:

~~~html
<h1>{{ t('hello', { name: user().name }) }}</h1>
<span>{{ t('cart.items', { count: cart.items().length }) }}</span>
~~~

~~~ts
import { t } from '@weave/i18n';

const greeting = computed(() => t('hello', { name: name() }));
~~~

The trick is the same one signals always play: called inside a tracked scope (a binding, a `computed`, an `effect`) `t` subscribes and re-runs on language change; called bare it just returns the current snapshot. One import — there's no `translate` vs `selectTranslate` split.

## Switching language

`setLocale(lang)` switches the active language, lazy-loading it (and the fallback) first, and resolves once it's active. Every `t` binding updates automatically:

~~~ts
import { setLocale, locale, loading } from '@weave/i18n';

<button on:click={{ () => setLocale('lt') }}>Lietuvių</button>
<span>Current: {{ locale() }}</span>
@if (loading()) { <Spinner /> }
~~~

`locale()` is the reactive active language and `loading()` is true while any load is in flight — both subscribe like any signal.

## ICU messages

Messages support ICU syntax for plurals, selection, numbers, and dates, all on native `Intl` (zero dependencies):

~~~
{count, plural, =0 {no files} one {# file} other {# files}}
{gender, select, male {he} female {she} other {they}}
{price, number, currency}
{when, date, medium}
~~~

So `t('cart.items', { count: 3 })` → `"3 items"`, and the same key with `count: 1` → `"1 item"` — and the German/Lithuanian/Arabic plural rules are picked up from `Intl` for those locales for free.

## Resolution order

Each call resolves: **active language → fallback language → `missing` handler** (which by default returns the key itself, so a missing translation is visible, not blank). `has(key)` tells you reactively whether a key currently resolves.

## Lazy loading and scopes

For larger apps, don't ship every translation up front. Provide a `loader` and Weave fetches languages — and optional feature **scopes** within a language — on demand, tracking each so it never re-loads:

~~~ts
const i18n = createI18n({
  lang: 'en',
  loader: (lang, scope) =>
    import(`./locales/${lang}${scope ? '/' + scope : ''}.json`).then((m) => m.default),
});

await i18n.load('lt', 'checkout'); // warm a scope without switching to it
~~~

`i18n.scoped('checkout')` returns a `t` whose keys are prefixed with `checkout.` — handy for feature-scoped message files.

## Per-subtree overrides

The bare `t` uses the global instance, but you can override it for a subtree by providing a different instance through context — the [provide/inject](/learn/lifecycle-context-di#context-values-without-prop-drilling) mechanism. A context-provided instance wins over the global within that subtree:

~~~ts
import { I18nContext } from '@weave/i18n';
import { provide } from '@weave/runtime';

provide(I18nContext, standaloneInstance); // this subtree's `t` uses it
~~~

This is useful for an embedded widget that ships its own messages, or for previewing a locale in one panel without switching the whole app.

:::callout info "What you just learned"
`createI18n()` sets up signal-backed translation (optional — unused, it costs nothing). `t(key, params)` works the same in templates and `.ts`, subscribing inside tracked scopes so bindings re-translate on `setLocale()` — with no reload. ICU plural/select/number/date ride native `Intl`. Resolution is active → fallback → `missing`. Lazy-load languages and scopes via `loader`; override per-subtree via `I18nContext`.
:::
