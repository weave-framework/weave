# Internationalization

`@weave/i18n` is signal-native translation. It's an *optional* add-on ‚Äî the runtime never depends on it, so an app that doesn't call `createI18n()` pays nothing. When you do use it, translation is just another signal graph: `t('key')` reads the active-locale and message signals, so every `{{ t('‚Ä¶') }}` (and every `computed`/`effect` over `t`) updates the instant `setLocale()` runs ‚Äî no reload, no streams, no RxJS.

The whole thing is zero-dependency: plurals, selects, numbers and dates all ride native `Intl`. There's no `messageformat` library underneath ‚Äî just a small in-house ICU parser.

## Setup

You create one instance, usually at startup. By default it registers itself as the *global* one, so a bare `import { t } from '@weave/i18n'` resolves to it everywhere ‚Äî no prop-drilling, no provider needed.

~~~ts title="app/i18n.ts"
import { createI18n } from '@weave/i18n';

export const i18n = createI18n({
  lang: 'en',
  fallbackLang: 'en',
  messages: {
    en: {
      hello: 'Hello, {{ name }}!',
      cart: { items: '{count, plural, =0 {empty} one {# item} other {# items}}' },
    },
    lt: {
      hello: 'Sveikas, {{ name }}!',
      cart: { items: '{count, plural, =0 {tu≈°ƒçias} one {# prekƒó} few {# prekƒós} other {# preki≈≥}}' },
    },
  },
});
~~~

Every field is optional except `lang`:

| Config | Type | Meaning |
|--------|------|---------|
| `lang` | `string` | Initial active language. The only required field. |
| `fallbackLang` | `string?` | Consulted when a key is missing in the active language. |
| `messages` | `Record<string, Messages>?` | Eagerly-available messages, keyed by language. May be nested any depth. |
| `langs` | `string[]?` | Advertised languages for a switcher. Defaults to the keys of `messages`. |
| `loader` | `Loader?` | Lazy loader for languages/scopes not present in `messages`. See below. |
| `missing` | `(key, lang) => string` | Called when a key resolves nowhere. Default: returns the key itself. |
| `global` | `boolean?` | Register as the global instance for the bare `t`. Default `true`. |

Messages are a tree of strings ‚Äî flat or nested, your call:

~~~ts title="flat vs nested ‚Äî both work"
// nested: look up with a dotted key, t('cart.items')
{ cart: { items: '‚Ä¶', total: '‚Ä¶' } }
// flat: look up with the literal key, t('cart.items')
{ 'cart.items': '‚Ä¶' }
~~~

`t('cart.items')` finds either. Lookup checks the literal key first, then walks the dots ‚Äî so a flat `'cart.items'` key wins over a nested `cart.items` if you somehow have both.

## Translating

`t(key, params?)` looks up a key and interpolates `params`. The same `t` works in templates **and** in plain `.ts` ‚Äî there's no `translate` vs `selectTranslate` split:

~~~html title="in a template"
<h1>{{ t('hello', { name: user().name }) }}</h1>
<span>{{ t('cart.items', { count: cart.items().length }) }}</span>
~~~

~~~ts title="in plain TypeScript"
import { t } from '@weave/i18n';

const greeting = computed(() => t('hello', { name: name() }));
~~~

The trick is the one signals always play: called inside a tracked scope (a binding, a `computed`, an `effect`) `t` subscribes to the locale + message signals and re-runs on language change; called bare it just returns the current snapshot.

### When a key isn't found

Resolution per call is **active language ‚Üí fallback language ‚Üí `missing` handler**. If a key resolves nowhere, the default `missing` returns the key string itself ‚Äî so a forgotten translation shows up as `cart.items` on screen rather than a blank, which is much easier to spot. Override it to do something else (log, throw, return a placeholder):

~~~ts title="custom missing handler"
createI18n({
  lang: 'en',
  missing: (key, lang) => {
    console.warn(`[i18n] missing ${key} for ${lang}`);
    return `‚ü®${key}‚ü©`;
  },
});
~~~

There's a fast path worth knowing about: if a found message is a plain string with **no `{`** and you pass **no `params`**, `t` returns it verbatim without running the ICU formatter at all. The moment a message contains `{` (or you pass params), it goes through `formatMessage`.

## Switching language

`setLocale(lang)` switches the active language and resolves once it's active. It does a bit of work first:

1. Lazy-loads the target language (if a `loader` is set and it isn't loaded yet).
2. Then loads the **fallback** language too ‚Äî but only if `fallbackLang` is set *and* differs from the target.
3. Then flips the active-language signal, which fires every `t` binding.

~~~ts title="a language switcher"
import { setLocale, locale, loading } from '@weave/i18n';
~~~

~~~html
<button on:click={{ () => setLocale('lt') }}>Lietuvi≈≥</button>
<span>Current: {{ locale() }}</span>
@if (loading()) { <Spinner /> }
~~~

`locale()` is the reactive active language; `loading()` is `true` while *any* language or scope load is in flight. Both subscribe like any signal, so the spinner appears and disappears on its own.

:::callout tip "First paint with a loader"
If you configure a `loader` (and no eager `messages` for the start language), `createI18n()` kicks off `setLocale(config.lang)` immediately at startup. That load is async, so the very first paint can show empty strings (the missing-key fallback) until it lands. Guard it with `@if (loading())`, or ship the start language eagerly in `messages` so it's there from frame one.
:::

## The exports

There are two layers: **bare exports** that talk to whatever the *active* instance is, and **instance methods** on what `createI18n()` returns.

### Bare exports

These resolve the active instance on every call ‚Äî a context-provided one wins, else the global. **They throw** if no instance exists yet (you never called `createI18n()`, or you're outside any `I18nContext` provider and there's no global). The error tells you to call `createI18n()` or provide one.

| Export | Signature | What it does |
|--------|-----------|--------------|
| `t` | `(key, params?) => string` | Translate via the active instance. |
| `locale` | `() => string` | Active language. Reactive. |
| `setLocale` | `(lang) => Promise<void>` | Switch language on the active instance. |
| `loading` | `() => boolean` | Whether the active instance is loading. Reactive. |
| `setGlobalI18n` | `(instance) => void` | Manually set the instance backing the bare exports. |
| `I18nContext` | `Context<I18n \| undefined>` | The context token for per-subtree overrides (see below). |

`setGlobalI18n` is the escape hatch for when you build an instance with `global: false` but later want it to be the global one anyway ‚Äî most apps never touch it, because `createI18n()` already registers itself unless you opt out.

### Instance methods

`createI18n()` returns an `I18n` object. These are the same operations the bare exports forward to, but called on a specific instance ‚Äî handy when you hold several instances or used `global: false`:

| Method | Signature | What it does |
|--------|-----------|--------------|
| `i18n.t` | `(key, params?) => string` | Translate a key. |
| `i18n.locale` | `() => string` | Active language. Reactive. |
| `i18n.availableLangs` | `() => string[]` | Languages this instance knows ‚Äî your `langs`, or the keys of loaded messages. Good for building a switcher. |
| `i18n.loading` | `() => boolean` | Whether any load is in flight. Reactive. |
| `i18n.setLocale` | `(lang) => Promise<void>` | Switch language (loads it + fallback first). |
| `i18n.load` | `(lang, scope?) => Promise<void>` | Load a language (and optional scope) **without** switching to it. |
| `i18n.has` | `(key) => boolean` | Whether `key` currently resolves (active or fallback). Reactive. |
| `i18n.scoped` | `(scope) => TranslateFn` | A `t` whose keys are prefixed with `scope.`. |

:::callout info "has() is an instance method, not a bare export"
There is no bare `import { has } from '@weave/i18n'`. To ask reactively whether a key resolves, call it on the instance: `i18n.has('cart.empty')`. (`availableLangs` and `load` are likewise instance-only.)
:::

## ICU messages

When a message contains `{`, it runs through the in-house ICU formatter. It's a compact subset of ICU MessageFormat ‚Äî enough for real apps, no dependency. Here's every form it understands.

### Interpolation

Two equivalent syntaxes for dropping a value in ‚Äî pick one and stay consistent:

~~~ts title="interpolation ‚Äî both are identical"
'Hello, {{ name }}!'   // double brace (transloco style)
'Hello, { name }!'     // single brace (ICU style)
~~~

Both read `params.name` and stringify it (`null`/`undefined` become `''`). Whitespace inside the braces is trimmed.

### Plural ‚Äî `{n, plural, ‚Ä¶}`

Cardinal plurals via `Intl.PluralRules`. The `#` inside a branch becomes the formatted number; `=N` matches an exact value and **wins over** the category match:

~~~ts title="plural"
'{count, plural, =0 {no files} one {# file} other {# files}}'
~~~

- `=N` (e.g. `=0`, `=1`) ‚Äî an exact-value branch, checked first.
- Category keywords ‚Äî `zero`, `one`, `two`, `few`, `many`, `other` ‚Äî chosen by `Intl.PluralRules` for the *active locale*. Lithuanian, Arabic, Russian, etc. pick the right one for free.
- `other` is the catch-all; include it.
- `#` inside the branch renders the number with `Intl.NumberFormat` for the locale.

So `t('cart.items', { count: 3 })` ‚Üí `"3 items"`, `count: 1` ‚Üí `"1 item"`, `count: 0` ‚Üí `"no files"` (the `=0` branch beats `other`).

### Ordinal ‚Äî `{n, selectordinal, ‚Ä¶}`

Same shape as `plural`, but uses ordinal rules (1st, 2nd, 3rd) instead of cardinal. `#` and `=N` work identically:

~~~ts title="selectordinal"
'{place, selectordinal, one {#st} two {#nd} few {#rd} other {#th}}'
~~~

`place: 1` ‚Üí `"1st"`, `2` ‚Üí `"2nd"`, `3` ‚Üí `"3rd"`, `4` ‚Üí `"4th"` (English ordinal categories from `Intl.PluralRules` with `type: 'ordinal'`).

### Select ‚Äî `{g, select, ‚Ä¶}`

A keyed branch ‚Äî the value is matched as a literal string against selectors, with `other` as the fallback:

~~~ts title="select"
'{gender, select, male {he} female {she} other {they}}'
~~~

If the value matches no selector and there's no `other`, the result is an empty string ‚Äî so always provide `other`.

### Number ‚Äî `{x, number[, style]}`

Formats with `Intl.NumberFormat` for the locale. Only two styles are recognized; anything else is plain number formatting:

| Form | Output |
|------|--------|
| `{x, number}` | Plain locale number formatting. |
| `{x, number, percent}` | Percent ‚Äî `0.25` ‚Üí `"25%"`. |
| `{x, number, integer}` | No fraction digits (`maximumFractionDigits: 0`). |

:::callout info "No `currency` style"
ICU's `{x, number, currency}` is **not** supported here. An unrecognized style (including `currency`) falls through to plain `Intl.NumberFormat` with no options ‚Äî so you get the number, but **no currency symbol**. To show money, format the value yourself with `Intl.NumberFormat(locale, { style: 'currency', currency: 'EUR' })` and pass the finished string in as a plain `{{ }}` param.
:::

### Date and time ‚Äî `{d, date|time[, style]}`

Formats a `Date` (or anything `new Date()` accepts ‚Äî a timestamp or ISO string) with `Intl.DateTimeFormat`:

~~~ts title="date / time"
'{when, date, medium}'   // dateStyle
'{when, time, short}'    // timeStyle
~~~

The style is one of `short` ¬∑ `medium` ¬∑ `long` ¬∑ `full`. An **unknown or omitted** style defaults to `medium`. `date` sets `dateStyle`; `time` sets `timeStyle`.

### Nesting and `#`

Plural/select/selectordinal sub-messages nest to any depth and can themselves contain `{{ }}`, `{ ‚Ä¶ }`, or `#`:

~~~ts title="nested"
'{count, plural, =0 {no items} other {# items for {{ name }}}}'
~~~

`#` is the active plural number, and it's substituted **only inside a plural/selectordinal sub-message**. At the top level (no enclosing plural) a `#` is just a literal `#` character.

### Literal braces ‚Äî apostrophe quoting

To put a literal `{` or `}` in a message, wrap it in single quotes; a doubled `''` is a literal apostrophe:

| In the message | Renders as |
|----------------|------------|
| `'{'` | `{` |
| `'}'` | `}` |
| `''` | `'` |
| `'{not a placeholder}'` | `{not a placeholder}` |

An unterminated opening quote just consumes the rest of the string as literal text.

## Lazy loading and scopes

For larger apps, don't ship every translation up front. Provide a `loader` and Weave fetches languages ‚Äî and optional feature **scopes** within a language ‚Äî on demand:

~~~ts title="lazy loader"
const i18n = createI18n({
  lang: 'en',
  fallbackLang: 'en',
  loader: (lang, scope) =>
    import(`./locales/${lang}${scope ? '/' + scope : ''}.json`).then((m) => m.default),
});

await i18n.load('lt', 'checkout'); // warm a scope without switching to it
~~~

A few things the loader does for you:

- **Dedup.** Each `(lang, scope)` pair is tracked by a key like `lt::checkout`, so a language or scope never loads or awaits twice ‚Äî even under concurrent calls. Languages you supplied eagerly in `messages` are pre-marked loaded.
- **No-loader case.** If there's no `loader`, `load()` just marks the pair loaded and returns; nothing is fetched.
- **Scope merge.** A scoped load slots its messages under `langMsgs[scope]`; an unscoped load merges into the language's top level. So a scope `'checkout'` becomes the `checkout.*` namespace.
- **`loading()` reflects it.** A `pending` counter goes up around each fetch, so `loading()` is `true` while anything is in flight.

`i18n.scoped('checkout')` returns a `t` whose keys are prefixed with `checkout.` ‚Äî pair it with scoped loads so each feature owns its own message file:

~~~ts title="scoped translate"
const tc = i18n.scoped('checkout');
tc('title');        // really t('checkout.title')
tc('total', { n }); // really t('checkout.total', { n })
~~~

## Per-subtree overrides

The bare `t` uses the global instance, but you can override it for a subtree by providing a different instance through context ‚Äî the [provide/inject](/learn/lifecycle-context-di#context-values-without-prop-drilling) mechanism. A context-provided instance wins over the global within that subtree.

This is the natural partner to `global: false`: build a standalone instance that *doesn't* clobber the global, then wire it into one subtree via `I18nContext`:

~~~ts title="standalone instance, scoped to a subtree"
import { createI18n, I18nContext } from '@weave/i18n';
import { provide } from '@weave/runtime';

// global: false ‚Üí does NOT become the app-wide instance
const widgetI18n = createI18n({
  lang: 'en',
  global: false,
  messages: { en: { title: 'Widget' }, de: { title: 'Modul' } },
});

provide(I18nContext, widgetI18n); // this subtree's bare `t` now uses widgetI18n
~~~

Handy for an embedded widget that ships its own messages, or for previewing a locale in one panel without switching the whole app. Outside any provider, the bare exports fall back to the global; if there's neither a provider nor a global, they throw.

:::callout info "What you just learned"
`createI18n(config)` sets up signal-backed translation (optional ‚Äî unused, it costs nothing); `lang` is the only required field. Bare exports ‚Äî `t`, `locale()`, `setLocale()`, `loading()`, `setGlobalI18n()` ‚Äî talk to the active instance and **throw** if there's none. Instance methods add `availableLangs()`, `load()`, `has()`, and `scoped()`. The same `t` works in templates and `.ts`, re-translating on `setLocale()` with no reload. Resolution is active ‚Üí fallback ‚Üí `missing`. The ICU subset covers `{{ }}`/`{ }` interpolation, `plural` (with `=N` and `#`), `selectordinal`, `select`, `number` (`percent`/`integer` ‚Äî **not** currency), and `date`/`time`, with apostrophe-quoted literal braces. Lazy-load languages and scopes via `loader` (deduped per `lang::scope`); override per-subtree via `I18nContext`.
:::
