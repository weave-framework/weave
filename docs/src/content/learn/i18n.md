# Internationalization

`@weave-framework/i18n` is signal-native translation. It's an *optional* add-on — the runtime never depends on it, so an app that doesn't call `createI18n()` pays nothing. When you do use it, translation is just another signal graph: `t('key')` reads the active-locale and message signals, so every `{{ t('…') }}` (and every `computed`/`effect` over `t`) updates the instant `setLocale()` runs — no reload, no streams, no RxJS.

The whole thing is zero-dependency: plurals, selects, numbers and dates all ride native `Intl`. There's no `messageformat` library underneath — just a small in-house ICU parser.

## Setup

You create one instance, usually at startup. By default it registers itself as the *global* one, so a bare `import { t } from '@weave-framework/i18n'` resolves to it everywhere — no prop-drilling, no provider needed.

~~~ts title="app/i18n.ts"
import { createI18n } from '@weave-framework/i18n';

export const i18n = createI18n({
  lang: 'en',
  fallbackLang: 'en',
  messages: {
    en: {
      hello: 'Hello, {{ name }}!',
      cart: { items: '{count, plural, =0 {empty} one {# item} other {# items}}' },
    },
    lt: {
      hello: 'Sveikas, {{ name }}!',
      cart: { items: '{count, plural, =0 {tuščias} one {# prekė} few {# prekės} other {# prekių}}' },
    },
  },
});
~~~

Every field is optional except `lang`:

| Config | Type | Meaning |
|--------|------|---------|
| `lang` | `string` | Initial active language. The only required field. |
| `fallbackLang` | `string?` | Consulted when a key is missing in the active language. |
| `messages` | `Record<string, Messages>?` | Eagerly-available messages, keyed by language. May be nested any depth. |
| `langs` | `string[]?` | Advertised languages for a switcher. Defaults to the keys of `messages`. |
| `loader` | `Loader?` | Lazy loader for languages/scopes not present in `messages`. See below. |
| `missing` | `(key, lang) => string` | Called when a key resolves nowhere. Default: returns the key itself. |
| `global` | `boolean?` | Register as the global instance for the bare `t`. Default `true`. |

Messages are a tree of strings — flat or nested, your call:

~~~ts title="flat vs nested — both work"
// nested: look up with a dotted key, t('cart.items')
{ cart: { items: '…', total: '…' } }
// flat: look up with the literal key, t('cart.items')
{ 'cart.items': '…' }
~~~

`t('cart.items')` finds either. Lookup checks the literal key first, then walks the dots — so a flat `'cart.items'` key wins over a nested `cart.items` if you somehow have both.

## Translating

`t(key, params?)` looks up a key and interpolates `params`. The same `t` works in templates **and** in plain `.ts` — there's no `translate` vs `selectTranslate` split:

:::tabs
~~~html title="in a template"
<h1>{{ t('hello', { name: user().name }) }}</h1>
<span>{{ t('cart.items', { count: cart.items().length }) }}</span>
~~~
~~~ts title="in plain TypeScript"
import { t } from '@weave-framework/i18n';

const greeting = computed(() => t('hello', { name: name() }));
~~~
:::

The trick is the one signals always play: called inside a tracked scope (a binding, a `computed`, an `effect`) `t` subscribes to the locale + message signals and re-runs on language change; called bare it just returns the current snapshot.

### When a key isn't found

Resolution per call is **active language → fallback language → `missing` handler**. If a key resolves nowhere, the default `missing` returns the key string itself — so a forgotten translation shows up as `cart.items` on screen rather than a blank, which is much easier to spot. Override it to do something else (log, throw, return a placeholder):

~~~ts title="custom missing handler"
createI18n({
  lang: 'en',
  missing: (key, lang) => {
    console.warn(`[i18n] missing ${key} for ${lang}`);
    return `⟨${key}⟩`;
  },
});
~~~

There's a fast path worth knowing about: if a found message is a plain string with **no `{`** and you pass **no `params`**, `t` returns it verbatim without running the ICU formatter at all. The moment a message contains `{` (or you pass params), it goes through `formatMessage`.

## Switching language

`setLocale(lang)` switches the active language and resolves once it's active. It does a bit of work first:

1. Lazy-loads the target language (if a `loader` is set and it isn't loaded yet).
2. Then loads the **fallback** language too — but only if `fallbackLang` is set *and* differs from the target.
3. Then flips the active-language signal, which fires every `t` binding.

:::tabs
~~~ts title="switcher.ts"
import { setLocale, locale, loading } from '@weave-framework/i18n';
~~~
~~~html title="switcher.html"
<button on:click={{ () => setLocale('lt') }}>Lietuvių</button>
<span>Current: {{ locale() }}</span>
@if (loading()) { <Spinner /> }
~~~
:::

`locale()` is the reactive active language; `loading()` is `true` while *any* language or scope load is in flight. Both subscribe like any signal, so the spinner appears and disappears on its own.

:::callout tip "First paint with a loader"
If you configure a `loader` (and no eager `messages` for the start language), `createI18n()` kicks off `setLocale(config.lang)` immediately at startup. That load is async, so the very first paint can show empty strings (the missing-key fallback) until it lands. Guard it with `@if (loading())`, or ship the start language eagerly in `messages` so it's there from frame one.
:::

## The exports

There are two layers: **bare exports** that talk to whatever the *active* instance is, and **instance methods** on what `createI18n()` returns.

### Bare exports

These resolve the active instance on every call — a context-provided one wins, else the global. **They throw** if no instance exists yet (you never called `createI18n()`, or you're outside any `I18nContext` provider and there's no global). The error tells you to call `createI18n()` or provide one.

| Export | Signature | What it does |
|--------|-----------|--------------|
| `t` | `(key, params?) => string` | Translate via the active instance. |
| `locale` | `() => string` | Active language. Reactive. |
| `setLocale` | `(lang) => Promise<void>` | Switch language on the active instance. |
| `loading` | `() => boolean` | Whether the active instance is loading. Reactive. |
| `setGlobalI18n` | `(instance) => void` | Manually set the instance backing the bare exports. |
| `I18nContext` | `Context<I18n \| undefined>` | The context token for per-subtree overrides (see below). |

`setGlobalI18n` is the escape hatch for when you build an instance with `global: false` but later want it to be the global one anyway — most apps never touch it, because `createI18n()` already registers itself unless you opt out.

### Instance methods

`createI18n()` returns an `I18n` object. These are the same operations the bare exports forward to, but called on a specific instance — handy when you hold several instances or used `global: false`:

| Method | Signature | What it does |
|--------|-----------|--------------|
| `i18n.t` | `(key, params?) => string` | Translate a key. |
| `i18n.locale` | `() => string` | Active language. Reactive. |
| `i18n.availableLangs` | `() => string[]` | Languages this instance knows — your `langs`, or the keys of loaded messages. Good for building a switcher. |
| `i18n.loading` | `() => boolean` | Whether any load is in flight. Reactive. |
| `i18n.setLocale` | `(lang) => Promise<void>` | Switch language (loads it + fallback first). |
| `i18n.load` | `(lang, scope?) => Promise<void>` | Load a language (and optional scope) **without** switching to it. |
| `i18n.has` | `(key) => boolean` | Whether `key` currently resolves (active or fallback). Reactive. |
| `i18n.scoped` | `(scope) => TranslateFn` | A `t` whose keys are prefixed with `scope.`. |

:::callout info "has() is an instance method, not a bare export"
There is no bare `import { has } from '@weave-framework/i18n'`. To ask reactively whether a key resolves, call it on the instance: `i18n.has('cart.empty')`. (`availableLangs` and `load` are likewise instance-only.)
:::

## ICU messages

When a message contains `{`, it runs through the in-house ICU formatter. It's a compact subset of ICU MessageFormat — enough for real apps, no dependency. Here's every form it understands.

### Interpolation

Two equivalent syntaxes for dropping a value in — pick one and stay consistent:

~~~ts title="interpolation — both are identical"
'Hello, {{ name }}!'   // double brace (transloco style)
'Hello, { name }!'     // single brace (ICU style)
~~~

Both read `params.name` and stringify it (`null`/`undefined` become `''`). Whitespace inside the braces is trimmed.

### Plural — `{n, plural, …}`

Cardinal plurals via `Intl.PluralRules`. The `#` inside a branch becomes the formatted number; `=N` matches an exact value and **wins over** the category match:

~~~ts title="plural"
'{count, plural, =0 {no files} one {# file} other {# files}}'
~~~

- `=N` (e.g. `=0`, `=1`) — an exact-value branch, checked first.
- Category keywords — `zero`, `one`, `two`, `few`, `many`, `other` — chosen by `Intl.PluralRules` for the *active locale*. Lithuanian, Arabic, Russian, etc. pick the right one for free.
- `other` is the catch-all; include it.
- `#` inside the branch renders the number with `Intl.NumberFormat` for the locale.

So `t('cart.items', { count: 3 })` → `"3 items"`, `count: 1` → `"1 item"`, `count: 0` → `"no files"` (the `=0` branch beats `other`).

### Ordinal — `{n, selectordinal, …}`

Same shape as `plural`, but uses ordinal rules (1st, 2nd, 3rd) instead of cardinal. `#` and `=N` work identically:

~~~ts title="selectordinal"
'{place, selectordinal, one {#st} two {#nd} few {#rd} other {#th}}'
~~~

`place: 1` → `"1st"`, `2` → `"2nd"`, `3` → `"3rd"`, `4` → `"4th"` (English ordinal categories from `Intl.PluralRules` with `type: 'ordinal'`).

### Select — `{g, select, …}`

A keyed branch — the value is matched as a literal string against selectors, with `other` as the fallback:

~~~ts title="select"
'{gender, select, male {he} female {she} other {they}}'
~~~

If the value matches no selector and there's no `other`, the result is an empty string — so always provide `other`.

### Number — `{x, number[, style]}`

Formats with `Intl.NumberFormat` for the locale. Only two styles are recognized; anything else is plain number formatting:

| Form | Output |
|------|--------|
| `{x, number}` | Plain locale number formatting. |
| `{x, number, percent}` | Percent — `0.25` → `"25%"`. |
| `{x, number, integer}` | No fraction digits (`maximumFractionDigits: 0`). |

:::callout info "No `currency` style in ICU — use `formatCurrency`"
ICU's `{x, number, currency}` is **not** supported inside messages. Use the standalone
[`formatCurrency`](#standalone-formatters-the-pipes-replacement) formatter instead —
`{{ formatCurrency(9.9, 'EUR') }}` → `"€9.90"` — and pass a finished string into a message param if you need it inside one.
:::

### Date and time — `{d, date|time[, style]}`

Formats a `Date` (or anything `new Date()` accepts — a timestamp or ISO string) with `Intl.DateTimeFormat`:

~~~ts title="date / time"
'{when, date, medium}'   // dateStyle
'{when, time, short}'    // timeStyle
~~~

The style is one of `short` · `medium` · `long` · `full`. An **unknown or omitted** style defaults to `medium`. `date` sets `dateStyle`; `time` sets `timeStyle`.

### Nesting and `#`

Plural/select/selectordinal sub-messages nest to any depth and can themselves contain `{{ }}`, `{ … }`, or `#`:

~~~ts title="nested"
'{count, plural, =0 {no items} other {# items for {{ name }}}}'
~~~

`#` is the active plural number, and it's substituted **only inside a plural/selectordinal sub-message**. At the top level (no enclosing plural) a `#` is just a literal `#` character.

### Literal braces — apostrophe quoting

To put a literal `{` or `}` in a message, wrap it in single quotes; a doubled `''` is a literal apostrophe:

| In the message | Renders as |
|----------------|------------|
| `'{'` | `{` |
| `'}'` | `}` |
| `''` | `'` |
| `'{not a placeholder}'` | `{not a placeholder}` |

An unterminated opening quote just consumes the rest of the string as literal text.

## Lazy loading and scopes

For larger apps, don't ship every translation up front. Provide a `loader` and Weave fetches languages — and optional feature **scopes** within a language — on demand:

~~~ts title="lazy loader"
const i18n = createI18n({
  lang: 'en',
  fallbackLang: 'en',
  loader: (lang, scope) =>
    import(`./locales/${lang}${scope ? '/' + scope : ''}.json`).then((m) => m.default),
});

await i18n.load('lt', 'checkout'); // warm a scope without switching to it
~~~

A few things the loader does for you:

- **Dedup.** Each `(lang, scope)` pair is tracked by a key like `lt::checkout`, so a language or scope never loads or awaits twice — even under concurrent calls. Languages you supplied eagerly in `messages` are pre-marked loaded.
- **No-loader case.** If there's no `loader`, `load()` just marks the pair loaded and returns; nothing is fetched.
- **Scope merge.** A scoped load slots its messages under `langMsgs[scope]`; an unscoped load merges into the language's top level. So a scope `'checkout'` becomes the `checkout.*` namespace.
- **`loading()` reflects it.** A `pending` counter goes up around each fetch, so `loading()` is `true` while anything is in flight.

`i18n.scoped('checkout')` returns a `t` whose keys are prefixed with `checkout.` — pair it with scoped loads so each feature owns its own message file:

~~~ts title="scoped translate"
const tc = i18n.scoped('checkout');
tc('title');        // really t('checkout.title')
tc('total', { n }); // really t('checkout.total', { n })
~~~

## Per-subtree overrides

The bare `t` uses the global instance, but you can override it for a subtree by providing a different instance through context — the [provide/inject](/learn/lifecycle-context-di#context-values-without-prop-drilling) mechanism. A context-provided instance wins over the global within that subtree.

This is the natural partner to `global: false`: build a standalone instance that *doesn't* clobber the global, then wire it into one subtree via `I18nContext`:

~~~ts title="standalone instance, scoped to a subtree"
import { createI18n, I18nContext } from '@weave-framework/i18n';
import { provide } from '@weave-framework/runtime';

// global: false → does NOT become the app-wide instance
const widgetI18n = createI18n({
  lang: 'en',
  global: false,
  messages: { en: { title: 'Widget' }, de: { title: 'Modul' } },
});

provide(I18nContext, widgetI18n); // this subtree's bare `t` now uses widgetI18n
~~~

Handy for an embedded widget that ships its own messages, or for previewing a locale in one panel without switching the whole app. Outside any provider, the bare exports fall back to the global; if there's neither a provider nor a global, they throw.

## Standalone formatters (the pipes replacement)

Weave has no template pipes — you call a function. So Angular's `{{ v | currency:'EUR' }}` becomes `{{ formatCurrency(v, 'EUR') }}`. These thin `Intl` wrappers cover the everyday cases, in the template or in `.ts`:

~~~ts
import { formatNumber, formatCurrency, formatPercent, formatDate, formatRelativeTime, formatList } from '@weave-framework/i18n';

formatNumber(1234.5, { minimumFractionDigits: 2 }); // "1,234.50"
formatCurrency(9.9, 'EUR');                          // "€9.90"
formatPercent(0.42);                                 // "42%"     (value is a ratio)
formatDate(order.createdAt, { dateStyle: 'medium' });// "Jul 6, 2026"
formatRelativeTime(-3, 'day');                       // "3 days ago"
formatList(['a', 'b', 'c']);                         // "a, b, and c"
~~~

- **Locale.** Each takes an optional last `locale` argument. When omitted it uses the **active i18n instance's locale — reactively**, so a formatter in a binding re-formats on `setLocale()`. With no `createI18n()` set up, it falls back to the runtime's default locale (and never throws — the formatters work without i18n).
- **Options** are the native `Intl` option bags (`Intl.NumberFormatOptions` / `DateTimeFormatOptions` / …), so `dateStyle`/`timeStyle` (`'short'|'medium'|'long'|'full'`) give the Angular-preset feel, and number/currency get the full `Intl.NumberFormat` control.

| Angular pipe | Weave |
| --- | --- |
| `{{ n \| number:'1.2-2' }}` | `{{ formatNumber(n, { minimumFractionDigits: 2, maximumFractionDigits: 2 }) }}` |
| `{{ v \| currency:'EUR' }}` | `{{ formatCurrency(v, 'EUR') }}` |
| `{{ r \| percent }}` | `{{ formatPercent(r) }}` |
| `{{ d \| date:'medium' }}` | `{{ formatDate(d, { dateStyle: 'medium' }) }}` |

:::callout info "What you just learned"
`createI18n(config)` sets up signal-backed translation (optional — unused, it costs nothing); `lang` is the only required field. Bare exports — `t`, `locale()`, `setLocale()`, `loading()`, `setGlobalI18n()` — talk to the active instance and **throw** if there's none. Instance methods add `availableLangs()`, `load()`, `has()`, and `scoped()`. The same `t` works in templates and `.ts`, re-translating on `setLocale()` with no reload. Resolution is active → fallback → `missing`. The ICU subset covers `{{ }}`/`{ }` interpolation, `plural` (with `=N` and `#`), `selectordinal`, `select`, `number` (`percent`/`integer` — **not** currency), and `date`/`time`, with apostrophe-quoted literal braces. Lazy-load languages and scopes via `loader` (deduped per `lang::scope`); override per-subtree via `I18nContext`. Standalone `Intl` formatters — `formatNumber` / `formatCurrency` / `formatPercent` / `formatDate` / `formatRelativeTime` / `formatList` — replace Angular's pipes; they default to the active (reactive) locale and never throw when i18n isn't set up.
:::
